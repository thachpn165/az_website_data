## Câu hỏi thường gặp

AZ Cloud Storage có tương thích S3 không? __

Có. Tương thích hoàn toàn S3 API - bạn có thể dùng AWS SDK, AWS CLI, và các tools S3 như rclone, s3cmd, Cyberduck mà không cần thay đổi code.

Dữ liệu được bảo vệ như thế nào? __

Mỗi dữ liệu được replicate 3 bản trên các node độc lập và mã hóa khi lưu trữ (at rest) và truyền tải (in transit). Nếu một node gặp sự cố, dữ liệu vẫn an toàn và hệ thống tự động phục hồi. Cam kết 99.99% durability.

Có giới hạn dung lượng không? __

Không giới hạn tổng dung lượng. Mỗi object tối đa 5TB, tối đa 100 buckets. Dung lượng tự động mở rộng theo nhu cầu.

AZ Cloud Storage có hỗ trợ CDN không? __

Hiện tại AZDIGI chưa hỗ trợ CDN tích hợp để sử dụng AZ Cloud Storage với domain riêng. Tuy nhiên, bạn có thể sử dụng dịch vụ CDN bên ngoài như BunnyCDN, Cloudflare hoặc KeyCDN để cấu hình CDN cho S3 bucket. Các dịch vụ này hỗ trợ S3-compatible storage và cho phép tuỳ chỉnh tên miền riêng (custom domain) để phân phối nội dung.

Tích hợp S3 vào WordPress như thế nào? __

Sử dụng plugin WP Offload Media hoặc Media Cloud để kết nối WordPress với AZ Cloud Storage. Chỉ cần nhập Access Key, Secret Key và Endpoint. Ảnh, video upload sẽ tự động lưu vào S3, giảm tải server hosting và tăng tốc website.

Lợi ích khi dùng S3 cho WordPress là gì? __

Giảm dung lượng hosting, tăng tốc load ảnh với băng thông cao, dễ backup và migrate, không lo hết dung lượng server. Website chạy nhẹ hơn vì media được phục vụ trực tiếp từ Object Storage.

Cấu hình n8n sử dụng S3 storage như thế nào? __

Trong file .env của n8n, cấu hình N8N _EXTERNAL_ STORAGE _S3_ BUCKET, N8N _EXTERNAL_ STORAGE _S3_ ACCESS _KEY, N8N_ EXTERNAL _STORAGE_ S3 _SECRET_ KEY và Endpoint. File đính kèm từ email, form, webhook sẽ lưu vào S3 thay vì database, giúp n8n chạy ổn định hơn.

Dùng Nextcloud với AZ Cloud Storage được không? __

Có. Cấu hình Primary Storage hoặc External Storage trong Nextcloud để sử dụng S3. Bạn có thể xây dựng hệ thống cloud storage riêng như Google Drive nhưng toàn quyền kiểm soát dữ liệu, không phụ thuộc bên thứ ba.

FileRun có hỗ trợ S3 không? __

Có. FileRun hỗ trợ S3 làm storage backend. Kết hợp với AZ Cloud Storage để tạo hệ thống quản lý file chuyên nghiệp, thay thế Dropbox với giao diện thân thiện và bảo mật cao.

Backup database MySQL/PostgreSQL lên S3 như thế nào? __

Sử dụng mysqldump hoặc pg_dump kết hợp với rclone hoặc aws-cli để upload file backup lên S3. Có thể tự động hóa bằng cron job. Dữ liệu được replicate 3 bản, an toàn hơn lưu trên server.

Rclone hoạt động với AZ Cloud Storage không? __

Có. Rclone hỗ trợ hoàn toàn S3-compatible storage. Cấu hình với lệnh 'rclone config', nhập Access Key, Secret Key và Endpoint. Dùng để sync, backup, mount như ổ đĩa trên máy tính.

Có thể dùng Restic backup lên S3 không? __

Có. Restic hỗ trợ S3 backend với mã hóa end-to-end. Cấu hình RESTIC_REPOSITORY với s3:endpoint/bucket-name. Backup incremental nhanh, tiết kiệm dung lượng, dễ khôi phục từng phiên bản.

Duplicati có hỗ trợ AZ Cloud Storage không? __

Có. Duplicati hỗ trợ S3-compatible storage với giao diện đồ họa dễ sử dụng. Thiết lập backup tự động, mã hóa AES-256, nén dữ liệu và lập lịch backup định kỳ.

Dùng S3 lưu trữ dataset AI/ML như thế nào? __

Upload dataset bằng aws-cli, boto3 (Python) hoặc rclone. S3 phù hợp cho dataset lớn với khả năng đọc song song (parallel reads). Nhiều framework ML như TensorFlow, PyTorch hỗ trợ đọc trực tiếp từ S3.

Có thể mount S3 như ổ đĩa không? __

Có. Dùng rclone mount hoặc s3fs để mount bucket như thư mục trên Linux/macOS. Phù hợp cho các ứng dụng cần đọc/ghi file trực tiếp mà không cần thay đổi code.

Làm sao backup cPanel/DirectAdmin lên S3? __

cPanel hỗ trợ S3 destination trong Backup Configuration. DirectAdmin dùng script kết hợp rclone. Backup tự động hàng ngày, lưu trữ an toàn ngoài server, dễ khôi phục khi cần.

Laravel/PHP có thể dùng S3 không? __

Có. Laravel Filesystem hỗ trợ S3 driver sẵn. Cấu hình trong config/filesystems.php với driver 's3', nhập credentials. Upload file bằng Storage::disk('s3')->put(). Flysystem PHP cũng hỗ trợ S3.

Node.js kết nối S3 như thế nào? __

Dùng AWS SDK for JavaScript (@aws-sdk/client-s3). Cấu hình endpoint, credentials và gọi các method như PutObject, GetObject. Tương thích hoàn toàn, không cần thay đổi code nếu đang dùng AWS S3.

Python boto3 có hoạt động với AZ Cloud Storage? __

Có. boto3 hỗ trợ S3-compatible storage. Thêm endpoint _url khi khởi tạo client. Tất cả operations như upload_ file, download _file, list_ objects đều hoạt động bình thường.

Có API để quản lý bucket không? __

Có. Sử dụng S3 API để tạo/xóa bucket, upload/download objects, quản lý permissions. Hỗ trợ đầy đủ aws-cli, SDK các ngôn ngữ, và tools như Cyberduck, S3 Browser.
